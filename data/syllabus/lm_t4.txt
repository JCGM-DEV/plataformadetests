¿Qué elemento se utiliza en un Schema XML (XSD) para definir tipos complejos que contienen otros elementos o atributos, y que permite especificar la secuencia u orden de aparición de sus hijos?
xs:all
xs:choice
*xs:sequence
xs:restriction, que define restricciones sobre tipos de datos existentes.
EXPL:xs:sequence indica que los elementos hijos deben aparecer en el orden exacto definido. xs:choice permite solo uno de los elementos. xs:all permite cualquier orden.
Los DTD (Document Type Definition) tienen varios problemas que los Schemas XML resuelven. ¿Cuál de los siguientes es un problema recurrente de los DTD?
Su capacidad para combinarse fácilmente con otros DTD y su soporte nativo para espacios de nombres XML
Su soporte robusto para la definición de tipos de datos complejos y la especificación del elemento raíz del documento
*Su ausencia de tipos de datos, la dificultad para acciones sencillas como ordenar elementos, y su limitada compatibilidad con los espacios de nombres XML
DTD tiene mayor capacidad expresiva que XSD para definir estructuras complejas.
EXPL:La opción correcta es Su ausencia de tipos de datos, la dificultad para acciones sencillas como ordenar elementos, y su limitada compatibilidad con los espacios de nombres XML.
¿Cuál de los siguientes atributos del elemento raíz xs:schema en un XSD se utiliza para indicar si los elementos deben llevar el espacio de nombres delante (qualified o unqualified)?
TargetNamespace
xmlns:alias
*elementFormDefault
xmlns, para declarar el espacio de nombres por defecto del documento.
EXPL:Un namespace XML es un URI que identifica un vocabulario XML. Evita conflictos cuando se combinan elementos de diferentes esquemas. Se declara con xmlns="URI" o xmlns:prefijo="URI".
¿Cuál de las siguientes afirmaciones describe mejor el objetivo principal de un Schema XML en comparación con un DTD?
Un Schema XML se centra exclusivamente en la sintaxis de las etiquetas XML, sin capacidad para definir tipos de datos o restricciones
Los Schemas XML están escritos en un lenguaje diferente a XML, lo que requiere analizadores y APIs distintos para su procesamiento
*Un Schema XML tiene como objetivo definir la estructura base de un documento XML, incluyendo elementos, atributos, tipos de datos, orden, ocurrencia y valores por defecto o fijos, mejorando las capacidades de los DTD
DTD y XSD son equivalentes en capacidad expresiva y solo difieren en la sintaxis.
EXPL:La opción correcta es Un Schema XML tiene como objetivo definir la estructura base de un documento XML, incluyendo elementos, atributos, tipos de datos, orden, ocurrencia y valores por defecto o fijos, mejorando las capacidades de los DTD.
Para definir un tipo de dato simple personalizado en un Schema XML, ¿qué estructura principal se utiliza?
xs:complexType
*xs:simpleType
xs:element
xs:complexType, para definir tipos que contienen elementos hijos o atributos.
EXPL:La respuesta es 'xs:simpleType'. "xs:complexType" y "xs:element" son incorrectos porque describen conceptos diferentes.
En XML Schema, ¿cuál de las siguientes facetas o restricciones se utiliza para especificar un rango de caracteres admitidos o una expresión regular para el valor de un elemento o atributo?
xs:Enumeration
xs:MaxLength
*xs:Pattern
xs:Restriction, para limitar los valores posibles de un tipo de dato existente.
EXPL:La opción correcta es xs:Pattern.
¿Cómo se asocia un documento XML con un Schema XML (XSD) cuando se utilizan explícitamente los nombres de los espacios de nombres en las etiquetas del documento XML?
Mediante el atributo noNamespaceSchemaLocation

Mediante el atributo targetNamespace en el elemento raíz del documento XML.

En un documento XSD, si se desea especificar que un elemento puede aparecer un número ilimitado de veces, ¿qué valor se debe asignar al atributo maxOccurs?
0
1
*unbounded
El valor 0, que indica que el elemento no puede aparecer en el documento.
EXPL:maxOccurs="unbounded" permite que el elemento aparezca un número ilimitado de veces. minOccurs="0" lo hace opcional. minOccurs="1" (por defecto) lo hace obligatorio.

default
use con valor optional
*use con valor required
El atributo fixed, para establecer un valor fijo que no puede modificarse.
EXPL:La opción correcta es use con valor required.
Según los principios de validación de documentos XML, ¿cuál de las siguientes afirmaciones es correcta?
Un documento XML siempre es válido si está bien formado
*Un documento XML puede estar bien formado pero no ser válido si no cumple las restricciones de un DTD o XSD
La validación solo verifica la sintaxis del documento, no su estructura ni el tipo de datos
Un documento XML bien formado siempre es también válido según cualquier esquema.
EXPL:Un documento XML bien formado cumple las reglas sintácticas: un elemento raíz, todas las etiquetas cerradas, anidamiento correcto, atributos entre comillas y sensibilidad a mayúsculas.
