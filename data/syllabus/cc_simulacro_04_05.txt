SIMULACRO CLOUD COMPUTING — 04/05/2026

Una empresa tiene una aplicación web que recibe picos de tráfico solo en campañas puntuales (Black Friday). Quiere pagar lo mínimo posible el resto del año sin perder rendimiento en esos picos. ¿Qué solución es la MÁS adecuada?
A) Comprar servidores físicos para todo el año
B) Usar máquinas virtuales siempre activas
*C) Usar escalado automático
D) Usar almacenamiento en frío
EXPLICACION: El escalado automático permite ajustar recursos según la demanda, evitando pagar por capacidad innecesaria y garantizando rendimiento en picos.

Un desarrollador quiere desplegar una aplicación sin preocuparse por servidores, sistemas operativos ni infraestructura. Solo quiere subir código. ¿Qué modelo debe usar?
A) IaaS
B) PaaS
C) SaaS
*D) Serverless
EXPLICACION: Serverless abstrae completamente la infraestructura. El desarrollador solo gestiona el código y se ejecuta en función de eventos.

Una empresa necesita almacenar millones de imágenes accesibles desde internet mediante enlaces directos. ¿Qué tipo de almacenamiento es el más adecuado?
A) Bloques
B) Ficheros
*C) Objetos
D) RAM
EXPLICACION: El almacenamiento de objetos permite acceder a los datos mediante URL, ideal para contenido multimedia y web.

Un sistema requiere altísimo rendimiento en operaciones de lectura/escritura para una base de datos crítica. ¿Qué tipo de almacenamiento elegirías?
A) Objetos
*B) Bloques
C) Ficheros
D) Archivado
EXPLICACION: El almacenamiento en bloques ofrece alto rendimiento y baja latencia, ideal para bases de datos.

Un arquitecto diseña una aplicación en la nube con múltiples servidores en distintas zonas para evitar caídas. ¿Qué principio está aplicando?
A) Coste optimizado
B) Seguridad
*C) Fiabilidad
D) Escalabilidad
EXPLICACION: Distribuir recursos para evitar fallos mejora la tolerancia a fallos y la disponibilidad del sistema.

Una aplicación empieza a responder lentamente porque todos los usuarios acceden al mismo servidor. ¿Qué solución es más adecuada?
A) Aumentar almacenamiento
B) Eliminar monitorización
C) Reducir usuarios
*D) Implementar balanceador de carga
EXPLICACION: El balanceador distribuye el tráfico entre varios servidores, evitando sobrecargas.

Una startup necesita lanzar su aplicación rápidamente en distintos entornos (local, cloud, test) con el mismo comportamiento. ¿Qué tecnología usaría?
*A) Contenedores
B) Máquinas físicas
C) Bases de datos
D) DNS
EXPLICACION: Los contenedores garantizan portabilidad y consistencia entre entornos.

Una empresa quiere analizar relaciones complejas entre usuarios (por ejemplo, redes sociales). ¿Qué tipo de base de datos es más adecuada?
A) Relacional
B) Key-Value
C) Documental
*D) Grafos
EXPLICACION: Las bases de datos de grafos permiten representar relaciones mediante nodos y aristas.

Un sistema necesita detectar rápidamente si hay errores o caídas en producción para actuar automáticamente. ¿Qué componente es imprescindible?
A) Almacenamiento
*B) Monitorización
C) Virtualización
D) CDN
EXPLICACION: La monitorización permite detectar anomalías y activar alertas para reaccionar rápidamente.

Una empresa quiere reducir costes moviendo datos antiguos que casi no se usan, pero que deben conservarse. ¿Qué solución es más adecuada?
A) RAM
B) Base de datos
*C) Archivado
D) Bloques
EXPLICACION: El almacenamiento de archivado es barato y está pensado para datos a largo plazo con poco acceso.

¿Qué modelo implica pagar solo por lo que se usa?
A) Pago fijo
*B) Pago por uso
C) Pago anual
D) Pago único
EXPLICACION: El modelo de pago por uso (pay-as-you-go) es una característica fundamental de la nube: solo pagas por los recursos que consumes.

¿Qué base de datos usa tablas?
A) NoSQL
B) Documental
*C) Relacional
D) Grafos
EXPLICACION: Las bases de datos relacionales organizan la información en tablas con filas y columnas, usando SQL para consultarlas.

¿Qué ventaja clave tiene el almacenamiento de bloques?
*A) Alto rendimiento
B) Bajo coste
C) Portabilidad
D) Simplicidad
EXPLICACION: El almacenamiento en bloques ofrece alto rendimiento y baja latencia, siendo ideal para cargas de trabajo de I/O intensivo como bases de datos.

¿Qué significa ACID?
*A) Atomicidad, consistencia, aislamiento, durabilidad
B) Acceso, control, integración, datos
C) Aplicación, código, interfaz, datos
D) Ninguna de las anteriores
EXPLICACION: ACID define las propiedades que garantizan que las transacciones de bases de datos se procesen de manera fiable: Atomicidad (todo o nada), Consistencia, Aislamiento y Durabilidad.

¿Qué es redundancia?
A) Eliminar datos
*B) Duplicar componentes
C) Reducir servidores
D) Aumentar coste
EXPLICACION: La redundancia consiste en duplicar componentes críticos del sistema para garantizar disponibilidad ante fallos.

¿Qué almacenamiento es ideal para bases de datos?
A) Objetos
B) Ficheros
*C) Bloques
D) Archivado
EXPLICACION: El almacenamiento en bloques ofrece el rendimiento y la baja latencia necesarios para sistemas de gestión de bases de datos.

¿Qué permite la computación en la nube?
*A) Acceder a recursos bajo demanda
B) Comprar más hardware
C) Reducir internet
D) Eliminar datos
EXPLICACION: La computación en la nube proporciona acceso bajo demanda a recursos de computación (servidores, almacenamiento, redes) a través de internet.

¿Qué característica permite aumentar recursos automáticamente?
A) Seguridad
*B) Escalabilidad
C) Latencia
D) Backup
EXPLICACION: La escalabilidad permite a un sistema aumentar (o disminuir) sus recursos de forma dinámica para adaptarse a la demanda.

¿Qué significa NoSQL?
*A) Not Only SQL
B) No SQL
C) New SQL
D) Next SQL
EXPLICACION: NoSQL significa "Not Only SQL" e indica que estas bases de datos no están limitadas al modelo relacional, pudiendo almacenar datos en formatos como documentos, grafos o clave-valor.

¿Qué permite el archivado de datos?
A) Eliminar datos
B) Aumentar velocidad
*C) Guardar a largo plazo
D) Reducir seguridad
EXPLICACION: El archivado de datos está diseñado para guardar información a largo plazo a bajo coste, aunque con mayor latencia de acceso.
